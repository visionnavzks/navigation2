# constrained_smoother 原理详解

本文面向这个独立版 `constrained_smoother` 的使用者和维护者，重点解释它是如何把一条离散路径转化为一个带约束的非线性最小二乘问题，并在求解后恢复为可执行的平滑路径。内容基于当前目录下的头文件实现，而不是上游 ROS 插件层的行为。

## 1. 模块目标

这个库解决的不是“重新规划路径”，而是“在尽量保留原路径拓扑和通行意图的前提下，对路径做连续几何优化”。它主要试图同时满足以下目标：

1. 让路径更平滑，减少折线感。
2. 保持曲率不超过车辆能力上限。
3. 不要偏离原始路径过远。
4. 利用 costmap 代价把路径从障碍物附近推开。
5. 在前进/倒车切换的尖点附近加强避障。

这意味着它不是一个单目标优化器，而是把多个目标写成若干残差项，再交给 Ceres 统一求解。

## 2. 核心数据约定

### 2.1 路径点的表示

输入路径类型是：

```cpp
std::vector<Eigen::Vector3d>
```

每个点的三个分量在不同阶段有不同语义：

1. 在 `smooth()` 输入阶段：
   - `x`、`y` 是世界坐标。
   - 第三个分量不是角度，而是运动方向符号，通常为 `+1` 或 `-1`。
2. 在 `upsampleAndPopulate()` 输出阶段：
   - `x`、`y` 仍然是世界坐标。
   - 第三个分量被改写为航向角 `yaw`，单位是弧度。

这是整个实现里最容易被忽略的隐含约定。如果调用方在输入时把第三维当成 yaw 传入，尖点检测和方向推断都会失真；如果调用方在输出后仍把第三维当方向符号使用，也会读错结果。

### 2.2 真正被优化的变量

虽然路径容器是 `Eigen::Vector3d`，但 Ceres 残差块只绑定每个点的前两个分量：

```cpp
ceres::AutoDiffCostFunction<SmootherCostFunction, 6, 2, 2, 2>
```

也就是说，每个残差块只把“前一点、当前点、后一点”的二维坐标作为参数块。第三维既不参与求导，也不参与求解，它只在路径扫描和后处理中提供方向语义。

因此，从优化器视角看，这个问题本质上是二维几何曲线优化，而不是 SE(2) 轨迹优化。

### 2.3 costmap 约定

`Costmap2D` 是对 ROS `Costmap2D` 的一个轻量替代，内部保存一块行优先的 `unsigned char` 栅格。优化器依赖以下信息：

1. 地图原点 `origin_x`、`origin_y`。
2. 栅格分辨率 `resolution`。
3. 原始代价值数组 `getCharMap()`。

代价查询不是用最近邻采样，而是通过 `ceres::Grid2D<unsigned char>` + `ceres::BiCubicInterpolator` 做双三次插值。因此，优化器看到的是连续位置上的平滑 cost field，而不是离散台阶函数，这对自动求导和收敛稳定性很重要。

## 3. 执行总流程

`Smoother::smooth()` 的总体流程可以拆成四步：

1. 校验输入路径长度，至少需要 2 个点。
2. 按当前参数构建 `ceres::Problem`。
3. 调用 Ceres 求解。
4. 按优化结果重新生成路径、补点并赋予姿态。

可以把这条链路概括为：

```text
原始离散路径
-> 下采样与尖点分析
-> 构造三点局部残差块
-> Ceres 非线性最小二乘
-> Bezier 上采样
-> 计算每个输出点的 yaw
```

## 4. 优化问题是如何建出来的

### 4.1 局部三点模型

优化不是一次性对整条路径写一个大公式，而是对路径上的每个“可优化中间点”建立一个局部三点残差：

1. `pt_prev`：前一个保留下来的路径点。
2. `pt`：当前被优化的中间点。
3. `pt_next`：后一个保留下来的路径点。

每个三点块会生成 6 维残差：

1. 平滑项 2 维。
2. 曲率项 1 维。
3. 距离原路径项 2 维。
4. costmap 项 1 维。

这也是 `AutoDiffCostFunction` 模板里的 `6` 的来源。

### 4.2 为什么是三点，而不是两点

如果只看两点，只能约束段长或朝向，无法表达“中间点是否形成尖锐转折”。三点模型可以直接描述离散二阶几何性质：

1. 平滑项通过前后差分的差异约束二阶变化。
2. 曲率项通过三点确定的圆弧半径近似局部曲率。
3. 切线方向也可以由三点恢复出来，为姿态估计和多检查点避障提供方向基。

这使整个问题既局部、又足够表达转弯几何。

## 5. 四类残差的数学含义

下面按 `SmootherCostFunction` 的实现解释四类代价项。

### 5.1 平滑残差 `addSmoothingResidual`

平滑项使用当前点前后两段向量：

1. `d_prev = pt - pt_prev`
2. `d_next = pt_next - pt`

然后构造：

```text
d_diff = ratio * d_next - d_prev
```

其中 `ratio = last_to_current_length_ratio_`，表示“上一段长度 / 当前段长度”，如果遇到需要把运动方向翻转考虑进来的情况，符号会被设成负值。

更精确地说，当前独立版代码里它是在构建残差块时按下面的表达式传入的：

```cpp
(last_was_cusp ? -1 : 1) * last_segment_len / current_segment_len
```

其中：

1. `last_segment_len` 是上一段保留路径的长度，也就是 `prelast_i -> last_i` 的二维距离。
2. `current_segment_len` 是当前段保留路径的长度，也就是 `last_i -> i` 的二维距离。
3. `last_was_cusp` 为真时，说明上一段与当前段之间跨过了前进/倒车切换点，此时比值整体取负号。

所以从当前实现的真实计算看：

$$
   ext{last\_to\_current\_length\_ratio\_} =
\pm \frac{\lVert \mathbf{p}_{last} - \mathbf{p}_{prelast} \rVert}{\lVert \mathbf{p}_{current} - \mathbf{p}_{last} \rVert}
$$

这里的符号取决于是否跨过 cusp。

现在变量名已经按真实实现修正为 `last_to_current_length_ratio_`，应理解为“上一段长度 / 当前段长度”，不要再按“next / last”去解读。

这一步的意义是把不等长线段归一到同一尺度上，再比较它们是否连续。如果两段在方向和相对尺度上衔接自然，`d_diff` 会接近 0；如果中间点形成尖角或局部摆动，残差就会上升。

与简单的 `pt_next - 2 * pt + pt_prev` 相比，这个实现多了一层长度比校正，因此对不均匀采样路径更稳一些。

### 5.2 曲率残差 `addCurvatureResidual`

曲率项的核心思想是：三点确定一个圆，圆半径越小，局部转弯越急。

实现步骤是：

1. 调用 `arcCenter(pt_prev, pt, pt_next, is_cusp)` 求过三点圆的圆心。
2. 若三点近似共线，则认为局部是直线，不产生曲率罚项。
3. 计算转弯半径 `turning_rad = |pt - center|`。
4. 计算超限量 `1 / turning_rad - max_curvature`。
5. 只有当超出允许最大曲率时才加罚。

因此它不是“鼓励曲率越小越好”，而是一个软约束：

1. 没超限，不罚。
2. 超限后，按超过的幅度线性增加残差。

这对移动机器人很重要，因为很多底盘并不要求路径完全近似直线，只要求不小于最小转弯半径即可。

### 5.3 距离原路径残差 `addDistanceResidual`

距离项最直接：

```text
diff = optimized_point - original_point
```

然后对 `x`、`y` 分别加权。

它的作用是给优化器一个“回拉”机制，避免在平滑项和避障项共同作用下偏离原路径太远。这个量可以理解为一种正则化：

1. `w_dist` 越大，越忠于原始几何。
2. `w_dist` 越小，优化器越敢为了避障或平滑去挪动节点。

### 5.4 costmap 残差 `addCostResidual`

避障项有两种工作模式。

#### 模式 A：单点采样

当 `cost_check_points` 为空时，只对路径点本身采样 costmap。

步骤是：

1. 把世界坐标减去地图原点。
2. 再除以分辨率，转为栅格浮点坐标。
3. 通过双三次插值读取该点附近的代价。
4. 用 `costmap_weight_sqrt` 加权。

这种模式适合近似点机器人或只想做轻量障碍回避的场景。

#### 模式 B：机器人轮廓多检查点采样

当 `cost_check_points` 非空时，向量按三元组解释：

```text
[x1, y1, weight1, x2, y2, weight2, ...]
```

每个 `(x, y)` 都是在机器人局部坐标系里的检查点。实现会先推导当前路径点的切线方向，再构造一个刚体变换，把这些局部点变换到世界系，逐个到 costmap 里采样并累加。

这个模式的意义是：

1. 不是只关心路径中心线是否靠近障碍。
2. 而是让机器人某些关键轮廓点也远离障碍。

对于细长、偏心、非圆形机器人，这比中心点采样更贴近真实碰撞风险。

需要注意的是，这个独立版只是读取 `cost_check_points` 并直接使用，没有像上游 ROS 版本那样对权重做开方和归一化，所以调用方需要自己保证权重尺度合理。

## 6. 尖点与倒车区是如何处理的

### 6.1 尖点的定义

在当前实现中，尖点不是通过角度突变判断，而是通过相邻保留点的方向符号是否翻转判断：

```text
is_cusp = current_direction_sign * last_direction_sign < 0
```

也就是说，只要路径第三维从 `+1` 变成 `-1`，或从 `-1` 变成 `+1`，就认为这里发生了前进/倒车切换。

### 6.2 为什么尖点需要特殊处理

前进和倒车的切换通常伴随以下问题：

1. 局部旋转更激烈。
2. 机器人外形相对障碍物的包络变化更危险。
3. 单纯的平滑项可能把轨迹往障碍物附近拉。

因此实现中会在尖点附近提高 costmap 代价权重。

### 6.3 权重不是突变，而是渐变

代码不是简单地“尖点前后若干个点乘一个固定系数”，而是按路径弧长做线性过渡。

实现方式大致如下：

1. 维护一个候选残差块队列 `potential_cusp_funcs`。
2. 队列中保存最近一段路径上的残差块及其段长。
3. 一旦发现尖点，就从尖点往前回溯，在 `cusp_zone_length / 2` 的范围内把 cost 权重从普通值平滑抬升到尖点值。
4. 尖点之后的一段路径则通过 `len_since_cusp` 继续用同样的线性策略逐渐降回普通权重。

因此 `cusp_zone_length` 对应的是“完整尖点区长度”，而尖点前后各占一半。

## 7. 下采样为什么这样做

### 7.1 基本目的

路径越长，残差块越多，Ceres 求解越慢。为此，构建问题时允许每隔若干个点只保留一个点进入优化。

### 7.2 不是简单固定间隔删点

实现的下采样有两个保护条件：

1. 起点和终点邻域的关键点会保留，以便维持起终姿态。
2. 如果某个点是尖点，不能因为下采样被跳过。

换句话说，`path_downsampling_factor` 只是“在安全前提下尽量少取点”，而不是机械抽样。

### 7.3 为什么要记录真实段长比

下采样后，相邻保留点之间可能跨过多个原始节点。因此局部前后两段长度不再均匀，平滑项也不能再假设单位步长。实现用：

```text
last_segment_len / current_segment_len
```

去修正前后段尺度，这就是 `last_to_current_length_ratio_` 的来源。

如果这一段还跨过 cusp，则同一个比例会额外乘上 `-1`，这样后续曲率中心和切线方向推导就会按“运动方向翻转”来解释这组三点关系。

## 8. 求解器层面的行为

### 8.1 求解器类型

`initialize()` 会把 `OptimizerParams` 写入 `ceres::Solver::Options`。默认线性求解器是：

```text
SPARSE_NORMAL_CHOLESKY
```

这对带稀疏结构的非线性最小二乘问题比较合适。

### 8.2 这是纯最小二乘，没有鲁棒核

构建残差块时：

```cpp
ceres::LossFunction * loss_function = NULL;
```

意味着当前实现没有对任何残差使用 Huber、Cauchy 之类的鲁棒损失。结果是：

1. 所有残差都会以标准平方和进入目标函数。
2. 极大 cost 或局部异常几何会产生更强烈的主导效应。

这让行为更直接，但也更依赖权重调参。

### 8.3 哪些点会被固定

为保持边界条件，以下点会被设为常量参数块：

1. 起点始终固定。
2. 终点始终固定。
3. 若 `keep_start_orientation = true`，第二个点也固定。
4. 若 `keep_goal_orientation = true`，倒数第二个点也固定。

这样做不是直接约束角度，而是通过固定相邻点位置间接保持端点切线方向。

### 8.4 什么情况下直接不优化

如果扣除固定点后没有剩余可优化点，`buildProblem()` 会返回 `false`，随后 `smooth()` 不执行 Ceres，而是直接进入后处理。这通常发生在：

1. 路径很短。
2. 下采样后只剩边界点。
3. 起终姿态保持导致中间自由度被吃光。

## 9. 求解后的路径是如何恢复的

优化完成后，并不会直接把下采样后的关键点作为最终输出，而是进入 `upsampleAndPopulate()` 做后处理。

### 9.1 起点姿态如何确定

起点输出姿态有两种来源：

1. 若 `keep_start_orientation = true`，直接使用调用方提供的 `start_dir`。
2. 否则，用起点到下一个优化点的方向结合运动符号估计。

### 9.2 中间关键点姿态如何确定

对每个已保留关键点，代码会用前后两个邻域点调用 `tangentDir()` 得到局部切线方向，然后根据后续运动方向修正符号，最后通过 `atan2()` 转成 yaw。

这意味着姿态并不是优化变量，而是由优化后的几何形状反推出来的派生量。

### 9.3 为什么要用三次 Bezier 上采样

如果前面做过下采样，关键点之间可能隔着多个原始节点。直接线性插值会重新引入折线感，所以实现使用分段三次 Bezier 曲线补点。

每段会构造四个控制点：

1. `pt0 = prelast_pt`
2. `pt3 = last_pt`
3. `pt1 = prelast_pt + prelast_dir * dist * 0.4 * sign`
4. `pt2 = last_pt - last_dir * dist * 0.4 * sign`

其中 `0.4` 是经验系数，用来控制切向拉伸程度；`sign` 来自运动方向，保证前进和倒车段的控制点方向一致。

再用标准三次 Bezier 公式在 `0..1` 之间插值补点。

### 9.4 `path_upsampling_factor` 的真实含义

这个参数容易望文生义，实际行为是：

1. `0`：不补点，输出保留下采样后的稀疏路径。
2. `1`：若前面做了每隔 `n` 点保留一个点，则这里大致补回原始离散粒度。
3. `2` 或更大：比原始粒度更密。

实现公式是：

```text
interp_cnt = (last_i - prelast_i) * path_upsampling_factor - 1
```

因此它和“关键点之间原来隔了多少个索引”直接相关，而不是一个固定的每段补点数。

### 9.5 插值点姿态如何赋值

Bezier 生成的中间点先只写入坐标，yaw 先置零。随后代码再遍历这些插值点，用相邻三个输出点计算切线方向，并按段方向修正正负，再写回 `atan2()` 角度。

## 10. 参数之间的联动关系

下面给出几个最常用参数的实际影响，而不是字面解释。

### 10.1 `smooth_weight_sqrt`

控制整体平滑趋势。

1. 太小：路径仍接近原始折线。
2. 太大：可能为了平滑而过度压缩局部细节，甚至与 cost 项对抗。

### 10.2 `curvature_weight_sqrt` 与 `max_curvature`

二者组合决定“超转弯能力”的惩罚强度。

1. `max_curvature` 决定红线在哪里。
2. `curvature_weight_sqrt` 决定越线后罚得有多重。

如果是差速或全向机器人，可以把 `max_curvature` 设大，或直接把曲率权重压低。

### 10.3 `distance_weight_sqrt`

决定是否允许路径明显偏离原轨迹。

1. 避障靠 costmap、又允许局部绕开时，可适当降低。
2. 如果上游规划器已经给出很可信的几何，只是想做轻微抛光，则应提高。

### 10.4 `costmap_weight_sqrt`

决定对环境障碍代价的敏感程度。

1. 太小：轨迹可能仍贴着障碍物走。
2. 太大：会强烈排斥高代价区，可能导致偏离原路径更多。

### 10.5 `cusp_costmap_weight_sqrt` 与 `cusp_zone_length`

决定尖点附近的特殊保护强度和作用范围。

1. 系数高但范围短：只在尖点附近强烈避障。
2. 系数中等但范围长：前后较大区间都被推离障碍。

### 10.6 `path_downsampling_factor` 和 `path_upsampling_factor`

这是效率与细节的直接杠杆。

1. 下采样增大，速度更快，但可能损失局部几何细节。
2. 上采样增大，输出更密，但计算的真实自由度并没有增加，只是后处理更细。

## 11. 当前独立版与上游 ROS 版本的一个重要差异

当前独立版 `options.hpp` 只是声明了参数结构，并不会像上游 ROS 版本那样自动：

1. 从参数服务器读取配置。
2. 把外部权重转换为平方根权重。
3. 对 `cost_check_points` 的权重做开方和归一化。

因此在这个独立版里，调用方需要自己保证：

1. 传入的各类 `*_weight_sqrt` 已经是平方根权重，而不是原始权重。
2. `max_curvature` 已经是曲率上限，而不是最小转弯半径。
3. `cost_check_points` 的第三列权重已经按自己需要预处理。

另外，`SmootherParams` 中保留了 `reversing_enabled` 字段，但当前独立版实现并未读取它。也就是说，当前版本始终假设输入路径第三维已经正确编码了前进/倒车方向，并据此做尖点检测。

## 12. 已知前提与限制

### 12.1 不支持只有姿态变化、位置不变化的连续点串

如果路径包含大量重复位置点，例如原地旋转段，三点几何和切线推断会退化。上游 README 也明确提示这类路径目前不受支持。

### 12.2 `costmap` 不能为空

当前实现会无条件访问 `costmap->getCharMap()` 等接口，因此调用方必须保证传入有效指针。

### 12.3 输入方向符号必须可信

尖点检测和部分切线符号修正都依赖第三维。如果方向符号错了，优化目标本身就会变。

### 12.4 解的可用性判断比较保守

求解后如果 `summary.IsSolutionUsable()` 为假，或者最终 cost 没有下降，代码会直接抛出 `FailedToSmoothPath`。这意味着它宁可拒绝结果，也不返回一个无法证明改进的路径。

## 13. 调参建议

如果落地时效果不理想，建议按下面顺序定位问题：

1. 先确认输入路径第三维真的是 `+1/-1` 方向符号，而不是 yaw。
2. 再确认 `*_weight_sqrt` 传入的是平方根权重。
3. 若路径贴障太近，先增大 `costmap_weight_sqrt`，再看是否需要启用 `cost_check_points`。
4. 若拐弯过急，先检查 `max_curvature` 是否设成了 `1 / minimum_turning_radius`。
5. 若求解太慢，先提高 `path_downsampling_factor`，再视需要用 `path_upsampling_factor` 恢复输出密度。
6. 若尖点附近仍危险，增大 `cusp_costmap_weight_sqrt` 或 `cusp_zone_length`。

## 14. 一句话总结实现本质

这个库本质上是在二维平面上，对一串带方向符号的离散路径节点建立局部三点残差，把“平滑、曲率约束、忠于原轨迹、远离障碍物”四种诉求拼成一个 Ceres 最小二乘问题，求解后再用切线和三次 Bezier 恢复成带姿态的连续可执行路径。
